# M6.6 — EngineStateChildEntry contract derivation cookbook

**Status:** developer reference for b.2–b.4 consumer migration off the live
`engine-statechild-parser.ts` and onto the native block stream.
**Companion to:** the b.1 SURVEY at
`docs/changes/m66-b1-native-contract-survey/progress.md` (commit `dfae2dab`),
the b.1 IMPL landing this file, AND the M6.6.b.1.5 dispatch
(`docs/changes/m66-b1-5-attr-tokenizer-extension/progress.md`) that extended
the native attribute tokenizer with `dotted-ident` / `wildcard` value kinds
and the `sourceText` verbatim-slice field. The recipes below assume the
b.1.5 tokenizer surface — pre-b.1.5 recipes are corrected in the
M6.6.b.1.5 CORRIGENDUM section below.
**Audience:** the b.2 (symbol-table.ts migration), b.3 (dependency-graph.ts /
component-3.ts migration), and b.4 (type-system migration) dispatch agents.

---

## Purpose

The b.1 SURVEY classified the 12-field live `EngineStateChildEntry` surface
(`compiler/src/symbol-table.ts:498-588`) into three buckets:

- **(a) already in native:** `tag` (1 field) — read directly off the block.
- **(b) derivable at consumer site:** 10 fields — recipes below.
- **(c) requires native extension:** `isColonShorthand` (1 field family) —
  landed by b.1 IMPL on `block.colonShorthandBody` (null = bare-body /
  self-close form; string = `:`-shorthand, verbatim post-`:` bytes).

This document gives the precise NATIVE-BLOCK WALK recipe for each (b) field
plus the discriminator recipe for the new (c) field. Each recipe is a small,
copy-pasteable shape spec so b.2–b.4 don't reinvent the wheel and stay
consistent on attribute-walk semantics.

## Conventions

Throughout this cookbook:

- `child` = a native Markup block from `engineBlock.children[]` representing
  one engine state-child opener (`<Variant ...>`).
- `engineBlock` = the native `<engine for=...>` Markup block (also
  `<machine ...>` for the legacy keyword).
- `source` = the full file source string (sliced to recover verbatim text).
- The native shape definitions live in
  `compiler/native-parser/tag-frame.js` (opener + Markup-block payload),
  `compiler/native-parser/parse-markup.js` (the block stream),
  `compiler/native-parser/collect-hoisted.js` (the engine routing).

### Shared block payload (native Markup block)

```
{
  kind:        "Markup",
  name:        string,                    // tag name (`tag` field)
  span:        { start, end, line, col }, // file-absolute opener-to-closer
  children:    Block[],                   // walkable body children
  closerForm:  "" | "</>" | "</Name>" | null,
  tagKind:     "Html" | "Component" | "ScrmlStructural" | "StateOpener",
  tagClass:    "Markup" | "Declaration" | "Compound" | "SelfClose" | "Structural",
  attrs:       AttrNode[],                // structured opener-attribute AST
  tokenizedAttrs: AttrTokenStream[],      // raw ATTR_* tokens

  // NEW (M6.6.b.1 — landed by b.1 IMPL):
  colonShorthandBody: string | null,      // null=bare/self-close; string=:-body
}
```

### Shared AttrNode shape

```
{
  name:  string,                          // attribute name
  value: AttrValue,                       // see kinds below
  span:  { start, end, line, col },       // file-absolute attribute span
}

AttrValue kinds (M6.6.b.1.5 — `dotted-ident` + `wildcard` added; every
non-`absent` variant ALSO carries a `sourceText: string` field — the
verbatim source slice INCLUDING any wrappers — recovers the original
form):
  { kind: "absent" }                                        // `<X foo>`        — no value (no sourceText)
  { kind: "string-literal", value: string, sourceText }     // `<X foo="bar">`  — `value` unquoted; `sourceText` keeps the quotes
  { kind: "variable-ref",  name: string,  sourceText }      // `<X foo=ident>` / `=@ident`
  { kind: "dotted-ident",  text: string,  sourceText }      // `<X foo=.Bar>`   — `text` keeps the leading dot (matches parseRuleAttrValue input)
  { kind: "wildcard",      text: "*",     sourceText }      // `<X foo=*>`      — standalone `*` token
  { kind: "expr",          raw: string,   refs, sourceText }// `<X foo=${expr}>` / `="..."` (if=) / `(...)` / `!...` / `[...]` / `{...}` (non-props)
                                                            //   `raw` is the UNWRAPPED expression; `sourceText` keeps any `${...}` / `"..."` / `{...}` wrapper
  { kind: "call-ref",      name, args, sourceText }         // `<X foo=fn(a,b)>` — `sourceText` includes the `(args)` wrapper
  { kind: "props-block",   propsDecl: string, sourceText }  // `<X props={...}>` — `propsDecl` inner; `sourceText` keeps the `{...}` wrapper
```

(Definitions at `compiler/native-parser/tag-frame.js:806-1156`; see the
header at L558 for the full attribute-tokenizer contract. M6.6.b.1.5
extensions added at L714+ — `isAttrDottedValueStart` /
`isAttrWildcardValueStart` predicates + the dispatch branches above the
`isAttrUnquotedValueStart` branch in `tokenizeAttributeRegion`.)

## Shared helpers (consumer-side, ~5 LOC each)

The b.2–b.4 consumers should hoist these to a small utility module
(e.g. `compiler/src/native-walker/engine-attrs.ts`) to avoid re-implementation.
They mirror the unexported helpers in `collect-hoisted.js:475-560` —
candidates for promotion to the native-parser's public surface if b.2+
needs them.

### `readAttrName(attrs, attrName) → string | null`

Returns the named attribute's value as an identifier-ish string, or `null`
when the attribute is absent / has no usable string value.

```typescript
function readAttrName(attrs: AttrNode[], attrName: string): string | null {
  for (const attr of attrs) {
    if (!attr || attr.name !== attrName) continue;
    const value = attr.value;
    if (!value) return null;
    if (value.kind === "variable-ref") return value.name;
    if (value.kind === "string-literal") return value.value;
    return null;
  }
  return null;
}
```

### `hasBareAttr(attrs, attrName) → boolean`

True iff the named attribute is present with an `absent` value (a bareword
modifier, e.g. `<X foo>` for `foo`).

```typescript
function hasBareAttr(attrs: AttrNode[], attrName: string): boolean {
  for (const attr of attrs) {
    if (!attr || attr.name !== attrName) continue;
    if (attr.value && attr.value.kind === "absent") return true;
  }
  return false;
}
```

### `readExprValue(attrs, attrName) → string | null`

Returns the UNWRAPPED expression text of an `expr`-kind attribute (the
`${`/`}` / `"..."` wrappers are stripped at the tokenizer; the value
carries just the inner expression text in `raw`). Used for
`effect=${...}` capture.

> **Cookbook factual fix (M6.6.b.1.5):** the pre-b.1.5 cookbook called
> this field `value.text` — that was wrong; the implementation has
> always used `value.raw` for the unwrapped expression text. The helper
> below uses the correct field name.

```typescript
function readExprValue(attrs: AttrNode[], attrName: string): string | null {
  for (const attr of attrs) {
    if (!attr || attr.name !== attrName) continue;
    if (attr.value && attr.value.kind === "expr") return attr.value.raw;
  }
  return null;
}
```

### `readDottedValue(attrs, attrName) → string | null` — M6.6.b.1.5

Returns the dotted-ident value's `text` field (which INCLUDES the leading
dot — matches `parseRuleAttrValue`'s input shape). Returns `null` when
the attribute is absent or has a non-dotted-ident value kind. Used as
the canonical reader for `rule=.X` / `internal:rule=.X` /
`initial=.X`-style state-variant attributes.

```typescript
function readDottedValue(attrs: AttrNode[], attrName: string): string | null {
  for (const attr of attrs) {
    if (!attr || attr.name !== attrName) continue;
    if (attr.value && attr.value.kind === "dotted-ident") return attr.value.text;
  }
  return null;
}
```

### `readWildcardValue(attrs, attrName) → boolean` — M6.6.b.1.5

Returns true iff the named attribute is present with a `wildcard`-kind
value (`rule=*`, `internal:rule=*`). Used as the canonical
wildcard-rule presence test.

```typescript
function readWildcardValue(attrs: AttrNode[], attrName: string): boolean {
  for (const attr of attrs) {
    if (!attr || attr.name !== attrName) continue;
    if (attr.value && attr.value.kind === "wildcard") return true;
  }
  return false;
}
```

### `readRuleAttrInput(attrs, attrName) → string | null` — M6.6.b.1.5

Returns the verbatim raw input that `parseRuleAttrValue` expects (the
union of the four legal `rule=` shapes: dotted-ident `.X`, wildcard
`*`, paren-form `(.A | .B)`, or quoted `"..."`). Routes each kind to
the correct accessor so the consumer can call
`parseRuleAttrValue(raw)` uniformly. Returns `null` when the attribute
is absent or has an unsupported value kind.

```typescript
function readRuleAttrInput(attrs: AttrNode[], attrName: string): string | null {
  for (const attr of attrs) {
    if (!attr || attr.name !== attrName) continue;
    const value = attr.value;
    if (!value) return null;
    // .X dotted-variant — `text` includes the leading dot.
    if (value.kind === "dotted-ident") return value.text;
    // * wildcard — the literal `*` is what parseRuleAttrValue expects.
    if (value.kind === "wildcard") return value.text;
    // (.A | .B) paren-form — `raw` includes the surrounding parens.
    if (value.kind === "expr") return value.raw;
    // ".X" quoted form — `value` is the unquoted content (no leading `"`).
    if (value.kind === "string-literal") return value.value;
    // X bare PascalCase form — parseRuleAttrValue accepts it defensively.
    if (value.kind === "variable-ref") return value.name;
    return null;
  }
  return null;
}
```

### `filterChildrenByName(children, name) → Block[]`

Returns the subset of children matching a name. Used for `<onTimeout>` /
`<onTransition>` sibling collection.

```typescript
function filterChildrenByName(children: Block[], name: string): Block[] {
  if (!Array.isArray(children)) return [];
  const out: Block[] = [];
  for (const c of children) {
    if (c && c.kind === "Markup" && c.name === name) out.push(c);
  }
  return out;
}
```

### `sliceFromSource(span, source) → string`

Verbatim source recovery. Used for `bodyRaw`, nested-engine `rawText`, etc.

```typescript
function sliceFromSource(span: Span, source: string): string {
  if (!span || typeof source !== "string") return "";
  return source.slice(span.start, span.end);
}
```

### `sliceBodyFromChildren(parent, source) → string`

The `collectRulesRaw` pattern — slice from the first child's `span.start` to
the last child's `span.end`. Used to recover a parent block's body region.

```typescript
function sliceBodyFromChildren(parent: Block, source: string): string {
  const children = parent?.children;
  if (!Array.isArray(children) || children.length === 0) return "";
  if (typeof source !== "string") return "";
  let lo = -1, hi = -1;
  for (const c of children) {
    const s = c?.span;
    if (!s) continue;
    if (lo < 0 || s.start < lo) lo = s.start;
    if (hi < 0 || s.end > hi) hi = s.end;
  }
  if (lo < 0 || hi < 0 || lo > hi || hi > source.length) return "";
  return source.slice(lo, hi);
}
```

## EngineStateChildEntry — field-by-field recipes

The live shape is at `compiler/src/symbol-table.ts:498-588`. Each recipe
below corresponds to one field; recipes are listed in declaration order so
b.2 can use this document as a direct migration checklist.

### `tag: string` — class (a)

The state-child's PascalCase tag name. Native already exposes this.

```typescript
const tag = child.name;
```

### `rule: EngineRuleForm` (6 kinds) — class (b)

The parsed `rule=` attribute. The live shape (symbol-table.ts:343-349) has 6
variants — `absent`, `single`, `multi`, `wildcard`, `legacy-arrow`,
`parse-error`. The parsing logic is in `parseRuleAttrValue` at
`compiler/src/engine-statechild-parser.ts` (look for the `parseRuleAttrValue`
export); the b.2 migration should import + reuse that pure helper.

```typescript
const ruleRaw = readRuleAttrInput(child.attrs, "rule");
const rule = ruleRaw !== null
  ? parseRuleAttrValue(ruleRaw)            // imported from engine-statechild-parser
  : { kind: "absent" } as EngineRuleForm;
```

**M6.6.b.1.5 — tokenizer surface (CORRECTED from pre-b.1.5 cookbook):**
the native attribute tokenizer admits four source forms for `rule=`:

| Source form          | AttrValue kind          | Accessor                                |
|----------------------|-------------------------|-----------------------------------------|
| `rule=.X`            | `dotted-ident`          | `value.text` — includes leading `.`     |
| `rule=*`             | `wildcard`              | `value.text` — the literal `"*"`        |
| `rule=(.A \| .B)`    | `expr`                  | `value.raw` — includes surrounding `()` |
| `rule=".X"` (quoted) | `string-literal`        | `value.value` — unquoted content        |
| `rule=X` (bare ident)| `variable-ref`          | `value.name` — defensive accept         |

`readRuleAttrInput` (above) routes all five into the verbatim string
that `parseRuleAttrValue` expects. The pre-b.1.5 cookbook's
`readAttrName`-based recipe was wrong for `dotted-ident` /
`wildcard` / `expr` kinds (returned null for all three — the STOP-doc
Gaps 1/2/3 regression). Use `readRuleAttrInput` instead.

### `bodyRaw: string` — class (b)

For the three opener forms:

- **`:`-shorthand:** `child.colonShorthandBody` directly (already the
  verbatim post-`:` text, leading whitespace stripped by b.1 IMPL).
- **bare-body** (`<Variant>...</>` / `<Variant>...</Variant>`):
  `sliceBodyFromChildren(child, source)` — the collectRulesRaw pattern.
- **self-close** (`<Variant/>`): `""` (empty body).

```typescript
let bodyRaw: string;
if (child.colonShorthandBody !== null) {
  bodyRaw = child.colonShorthandBody;
} else if (isSelfCloseShape(child)) {
  bodyRaw = "";
} else {
  bodyRaw = sliceBodyFromChildren(child, source);
}

// isSelfCloseShape: derived from child.closerForm + tagClass.
function isSelfCloseShape(child: Block): boolean {
  return child.tagClass === "SelfClose";
}
```

### `isColonShorthand: boolean` — class (c) — LANDED BY b.1 IMPL

Direct discriminator on the block:

```typescript
const isColonShorthand = child.colonShorthandBody !== null;
```

### `rawOffset: number` — class (b)

The substring offset of the state-child's opener `<` relative to the
enclosing engine's `rulesRaw` start (which is the engine body's first child
span.start, per `collectRulesRaw`).

```typescript
const engineRulesRawStart = computeRulesRawStart(engineBlock);
const rawOffset = child.span.start - engineRulesRawStart;

function computeRulesRawStart(engineBlock: Block): number {
  const children = engineBlock?.children;
  if (!Array.isArray(children) || children.length === 0) return 0;
  let lo = -1;
  for (const c of children) {
    if (c?.span && (lo < 0 || c.span.start < lo)) lo = c.span.start;
  }
  return lo < 0 ? 0 : lo;
}
```

### `historyAttr: boolean` — class (a/b)

A bareword `history` attribute on the opener.

```typescript
const historyAttr = hasBareAttr(child.attrs, "history");
```

### `internalRule: EngineRuleForm` (6 kinds) — class (b)

Same shape as `rule`, but reading `internal:rule=` instead of `rule=`.
The native attribute tokenizer admits `:` in attribute names (verified
in M6.6.b.1.5 — the tokenizer reads `internal:rule` as a single
attribute name token because the attribute-name scanner treats `:` as
a name-continue char). The value-side shapes are identical to `rule=`
(see the kind/accessor table above).

```typescript
const internalRuleRaw = readRuleAttrInput(child.attrs, "internal:rule");
const internalRule = internalRuleRaw !== null
  ? parseRuleAttrValue(internalRuleRaw)
  : { kind: "absent" } as EngineRuleForm;
```

**M6.6.b.1.5 verified:** OQ #1 (the b.1 cookbook open question about
`:`-in-name tokenization) is resolved — the parser-conformance test
`"internal:rule=.X" works identically — \`:\` in attr name does not
perturb dispatch"` confirms the attr name is one token + the
`dotted-ident` value is read identically to `rule=.X`.

### `onTimeoutElements: OnTimeoutEntry[]` — class (b)

Filter the state-child's body children for `<onTimeout>` siblings; build the
entry list. Skip regions for `<onTransition>` body + nested-engine body
(mirrors the live skip-regions logic at engine-statechild-parser.ts:1913+).

```typescript
const onTimeoutBlocks = filterChildrenByName(child.children, "onTimeout");
const bodyStart = computeBodyStart(child);  // for rawOffset arithmetic
const onTimeoutElements: OnTimeoutEntry[] = onTimeoutBlocks.map((b) => ({
  after: readExprValue(b.attrs, "after")
       ?? readAttrName(b.attrs, "after")
       ?? "",
  to: stripLeadingDot(readAttrName(b.attrs, "to") ?? ""),
  name: readAttrName(b.attrs, "name") ?? undefined,
  rawOffset: b.span.start - bodyStart,
}));

function computeBodyStart(child: Block): number {
  // For bare-body, body starts at first child's span.start.
  // For :-shorthand, body has no children — onTimeoutElements is [].
  // For self-close, ditto.
  const children = child.children;
  if (!Array.isArray(children) || children.length === 0) {
    return child.span.start;
  }
  let lo = -1;
  for (const c of children) {
    if (c?.span && (lo < 0 || c.span.start < lo)) lo = c.span.start;
  }
  return lo < 0 ? child.span.start : lo;
}
```

**Note:** the live's `onTimeoutElements` is `[]` for `:`-shorthand /
self-close forms. The check is `(isColonShorthand || isSelfClose) ? [] :
filter(...)` per engine-statechild-parser.ts:1955-1957.

### `innerEngines: NestedEngineEntry[]` — class (b)

Filter body children for nested `<engine>` blocks (uses the already-exported
`isEngineBlock` predicate from `collect-hoisted.js:344`).

```typescript
import { isEngineBlock } from "../../native-parser/collect-hoisted.js";

const innerEngineBlocks = (child.children ?? []).filter(isEngineBlock);
const innerEngines: NestedEngineEntry[] = innerEngineBlocks.map((b) => ({
  rawText: sliceFromSource(b.span, source),
  rawOffset: b.span.start - bodyStart,
}));
```

Same `:`-shorthand / self-close gating as `onTimeoutElements`.

### `effectRaw: string | null` — class (b)

The `effect=${...}` attribute on the state-child opener. The native attribute
tokenizer produces a `{ kind: "expr", raw: "..." }` value with the `${`/`}`
already stripped — directly readable. (M6.6.b.1.5: the field name is `raw`
NOT `text`; the pre-b.1.5 cookbook misnamed it.)

```typescript
const effectRaw = readExprValue(child.attrs, "effect");
```

**Compare to live:** the live engine-statechild-parser.ts:1798-1820 walks
the raw attribute text to find `effect=` then balances braces. The native
gives the same result with NO scanning — the attribute tokenizer's brace-
balanced `expr`-kind value IS the captured body.

### `onTransitionElements: OnTransitionEntry[]` — class (b)

Same pattern as `onTimeoutElements`, filtering for `<onTransition>`. The
OnTransitionEntry has 7 fields (`to`, `from`, `once`, `ifExprRaw`, `bodyRaw`,
`isColonShorthand`, `rawOffset`) — all derivable per the same recipes:

```typescript
const onTransitionBlocks = filterChildrenByName(child.children, "onTransition");
const onTransitionElements: OnTransitionEntry[] = onTransitionBlocks.map((b) => ({
  to:        stripLeadingDot(
               readDottedValue(b.attrs, "to")
            ?? readAttrName(b.attrs, "to")
             ) ?? null,
  from:      stripLeadingDot(
               readDottedValue(b.attrs, "from")
            ?? readAttrName(b.attrs, "from")
             ) ?? null,
  once:      hasBareAttr(b.attrs, "once"),
  // M6.6.b.1.5 — `ifExprRaw` legacy parity: the legacy parser preserved
  // the verbatim source slice (quotes for `if="..."`, wrapper for
  // `if=${...}`). Use `sourceText` to recover it; fall back to `raw`
  // / variable-ref name for unwrapped forms. See the `if=` recipe
  // section in the M6.6.b.1.5 CORRIGENDUM for the kind table.
  ifExprRaw: readIfExprRaw(b.attrs, "if"),
  bodyRaw: b.colonShorthandBody !== null
         ? b.colonShorthandBody
         : sliceBodyFromChildren(b, source),
  isColonShorthand: b.colonShorthandBody !== null,
  rawOffset: b.span.start - bodyStart,
}));

// readIfExprRaw — legacy `ifExprRaw` recovery using the M6.6.b.1.5
// `sourceText` field for verbatim parity (quoted form keeps `"..."`,
// `${...}` form keeps the wrapper).
function readIfExprRaw(attrs: AttrNode[], attrName: string): string | null {
  for (const attr of attrs) {
    if (!attr || attr.name !== attrName) continue;
    const value = attr.value;
    if (!value) return null;
    if (typeof value.sourceText === "string") return value.sourceText;
    if (value.kind === "expr") return value.raw;
    if (value.kind === "variable-ref") return value.name;
    return null;
  }
  return null;
}
```

Same `:`-shorthand / self-close gating as `onTimeoutElements`.

### `payloadBindings: PayloadBinding[]` — class (b)

Walk the opener attributes, skipping reserved names (`rule`, `effect`,
`history`, `internal:rule`) — what's left are payload bindings. The native
attribute value kind discriminates:

- `{ kind: "absent" }` (bareword): `{ kind: "positional", name: attr.name }`
- `{ kind: "variable-ref" }`: `{ kind: "named", field: attr.name, name: value.name }`

```typescript
const RESERVED_ATTRS = new Set(["rule", "effect", "history", "internal:rule"]);

const payloadBindings: PayloadBinding[] = [];
for (const attr of child.attrs) {
  if (!attr || RESERVED_ATTRS.has(attr.name)) continue;
  const value = attr.value;
  if (!value) continue;
  if (value.kind === "absent") {
    payloadBindings.push({ kind: "positional", name: attr.name });
  } else if (value.kind === "variable-ref") {
    payloadBindings.push({ kind: "named", field: attr.name, name: value.name });
  }
}
```

**Note:** the live's `parsePayloadBindings` at
`engine-statechild-parser.ts:1842` ALSO handles the parenthesized form
`<Done(rows)>`. M6.6.b.1.5 confirmed (STOP-doc Gap 4) that the native
attribute tokenizer's behavior for `<Done(rows)>` openers — both forms
(`<Done(rows)>` and `<Done rows>`) — yield identical attribute streams
because the `(` and `)` are silently consumed by the unexpected-char
skip branch. The `PayloadBinding` shape is recoverable from the
bareword `rows` attr in both cases, but the SOURCE FORM distinction
is lost. For B17.3-typer parity that requires source-form discrimination,
the consumer must scan the verbatim opener slice
(`sliceFromSource(child.span, source)`) — that's a known divergence
documented in OQ #2 below.

## Nested sub-types (recipes)

The nested-type field shapes are mostly recipe-equivalent to the
top-level recipes above (since they read the same attribute / body
shape from native blocks). Key per-type notes:

### `EngineRuleForm` (6 kinds)

Pure parser at `parseRuleAttrValue` in `engine-statechild-parser.ts` (import
+ reuse — no migration logic needed at b.2 site).

### `PayloadBinding` (2 kinds)

Covered by the `payloadBindings` recipe above. No additional dispatch.

### `OnTimeoutEntry` (4 fields — `after`, `to`, `name?`, `rawOffset`)

Each derivable from `onTimeoutBlock.attrs` + `onTimeoutBlock.span` per the
recipes above.

### `NestedEngineEntry` (2 fields — `rawText`, `rawOffset`)

`rawText` via `sliceFromSource`; `rawOffset` via span arithmetic. Already
shown above.

### `OnTransitionEntry` (7 fields)

Covered by the `onTransitionElements` recipe above. Note that
`isColonShorthand` on `<onTransition>` blocks uses the SAME
`block.colonShorthandBody` discriminator as state-child entries — the b.1
IMPL extension on `tokenizeOpener` applies uniformly to ALL openers, not
just state-children.

## M6.6.b.1.5 CORRIGENDUM SUMMARY

The b.1 cookbook (pre-M6.6.b.1.5) carried three classes of fact errors
that the M6.6.b.1.5 dispatch corrected:

| Cookbook claim                                          | Actual (M6.6.b.1.5)                           |
|---------------------------------------------------------|-----------------------------------------------|
| `expr` value carries `text: string`                     | Carries `raw: string` (always has — cookbook factual error) |
| `rule=.X` admits as `variable-ref`                      | Admits as `dotted-ident` (new kind, b.1.5)    |
| `rule=*` recoverable via `readAttrName`                 | Requires `readWildcardValue` (new kind, b.1.5) |
| `rule=(.A \| .B)` recoverable via `readAttrName`        | Requires `value.raw` from `expr` kind         |
| `internal:rule=.X` works (untested)                     | Works (M6.6.b.1.5 test confirms)              |
| `if=` quote/wrapper preservation (unspecified)          | Use `value.sourceText` (M6.6.b.1.5 new field) |

The corrected accessors are `readRuleAttrInput` (multi-form rule reader),
`readDottedValue` (dotted-ident reader), `readWildcardValue` (wildcard
presence test), and `readIfExprRaw` (legacy quote/wrapper-preserving
`ifExprRaw` recovery). All four are defined in the "Shared helpers"
section above.
