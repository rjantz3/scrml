# domain.map.md
# project: scrmlts
# updated: 2026-06-09T10:30:00Z  commit: 049954e0

The domain is the scrml COMPILER pipeline. scrml is a single-file, full-stack reactive web
language compiled by this TypeScript/JS toolchain running on Bun. The compiler converts `.scrml`
source files into `*.server.js` + `*.client.js` + `*.html` + `*.css` outputs.

## Business Invariants (from SPEC + code)
- `null` and `undefined` do NOT exist in scrml source; both â†’ `not` (SPEC Â§42; `W-ABSENCE-IN-SCRML-SOURCE`)
- `any` does NOT exist in scrml â€” the literal type-token `any` in ANY type position is `E-TYPE-ANY-FORBIDDEN` (Â§14.1.1, S174 hard line). The sanctioned untyped escape hatch is the named, greppable `asIs`. An arbitrary undefined type name that also resolves silently to `asIs` is a separate broader gap (deferred follow-on)
- Client JS MUST NOT contain SQL execution calls, server env access, or other server-only constructs (E-CG-006)
- `<auth role="X">` gates JS-mount only, NOT served HTML content (W-AUTH-CONTENT-NOT-GATED, GITI-027A)
- Every reactive write site must be in a logic context (E-WRITE-NOT-IN-LOGIC-CONTEXT)
- A reactive STATE CELL (plain Shape-1 or derived) may NOT be exported â€” `E-EXPORT-001` (Â§21.2, S173); a cell holds per-instance runtime state with no cross-file meaning. Components-as-const, `export <channel>`, and exported engines stay legal (keyed on `kind:"state-decl"`)
- A function is NOT value data â€” it may be PASSED (a prop, `W-COMPONENT-001`) or CALLED (handler), but NEVER STORED on a value-shaped collection. Storing one on a struct field is `E-STRUCT-FUNCTION-FIELD` (Â§14.3/Â§15.11, S175 hard line; the STORED face of the named passed-vs-stored rule Â§15.11.5.1; limit-the-primitive axiom Â§14.1.1). - Generated table types are NOMINAL and WALLED inside the `<db>` block (Â§14.8.1/Â§14.8.4). S175 carves ONE bounded exception (Â§14.8.8): a SQL projection row STRUCTURALLY width-subtypes into a developer-declared `:struct` prop/cell contract (one-directional: contract is a lower bound, EXTRA row columns OK). A miss is `E-SQL-ROW-CONTRACT-MISMATCH`. General struct-to-struct assignment stays nominal â€” a plain user struct coinciding with a row's columns is NOT width-subtype-assignable.
